# Security Policy

## Supported Version
- Saral Lang 1.0.0

## Reporting Vulnerabilities
Please report suspected security issues privately to the project maintainer before public disclosure.

## Security Goals
- Secure-by-design compiler architecture
- Input validation during lexing, parsing and semantic analysis
- Clear error reporting
- Safe handling of generated code

## Disclosure Process
1. Report the issue privately.
2. Allow time for verification and remediation.
3. Publish an advisory after a fix is available.

## Scope
This policy covers:
- Compiler
- CLI tools
- Installation scripts
- Standard project components
